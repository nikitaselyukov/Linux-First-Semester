## Ход работы

## 1. Базовые команды

### 1.1 Вывести все дочерние директории в `/proc`, игнорируя ошибки доступа

```sh
find /proc -mindepth 1 -type d 2>/dev/null
```

**Пояснение:**

- `find /proc` — выполняет обход каталога `/proc`.
- `-mindepth 1` — исключает из вывода сам каталог `/proc`, оставляя только вложенные элементы.
- `-type d` — ограничивает поиск только директориями.
- `2>/dev/null` — скрывает сообщения об ошибках доступа.

---

### 1.2 Вывести строки файла в обратном порядке

```sh
tac 1.1.sh
```

**Пояснение:**

- `tac` выводит содержимое файла построчно в обратной последовательности, то есть от последней строки к первой.

---

### 1.3 Показать список файлов текущей директории с правами (включая скрытые), директории — первыми

```sh
ls -lA --group-directories-first
```

**Пояснение:**

- `-l` — включает подробный формат вывода: права доступа, владельца, группу, размер и дату изменения.
- `-A` — показывает скрытые файлы, не выводя служебные записи `.` и `..`.
- `--group-directories-first` — размещает директории перед обычными файлами.

## 2. Файловая система sysfs

### 2.1 Считать параметр из `sysfs` (температура CPU / заряд батареи)

```sh
# Температура CPU
awk '{printf("%.1f°C\n",$1/1000)}' /sys/class/thermal/thermal_zone0/temp

# Уровень заряда батареи (0–100)
cat /sys/class/power_supply/BAT1/capacity
```

**Пояснение:**

- `sysfs` (`/sys`) — файловый интерфейс ядра Linux, предоставляющий доступ к параметрам устройств и подсистем.
- `awk` читает входные данные построчно и обрабатывает значения, разбивая строку на поля.
- Температура в `thermal_zone*/temp` обычно хранится в миллиградусах, поэтому значение делится на `1000`.
- Заряд батареи доступен в `BAT*/capacity` и выводится в процентах. На данной машине используется путь `/sys/class/power_supply/BAT1/capacity`.
