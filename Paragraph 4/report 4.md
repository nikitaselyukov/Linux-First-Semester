## Ход работы

## 1) Массовое переименование файлов (`vidir`)

### Файл

`01_mass_file_rename_with_vidir.txt`

### Решение (последовательность действий/клавиш)

```text
в директории:
env EDITOR=vim vidir

1) Esc
2) ggVG
3) g~
4) :wq + Enter
```

### Пояснение

- Команда `env EDITOR=vim vidir` открывает список файлов текущей директории в редакторе Vim.
- Нажатие `Esc` переводит редактор в нормальный режим.
- Комбинация `ggVG` выделяет весь список имён файлов: переход в начало документа, запуск построчного визуального режима и выделение до конца файла.
- Команда `g~` меняет регистр выделенного текста.
- Команда `:wq` сохраняет изменения и завершает работу редактора, после чего `vidir` применяет переименование к файлам.

### Пример запуска

```bash
cd "Paragraph 4"
env EDITOR=vim vidir
```

## 2) Массовая обработка строк (макрос: обернуть строку в кавычки)

### Файлы

- `02_quote_lines_with_vim_macro.txt`
- `03_sample_strings.txt`

### Тестовые данные

```text
hello world
 a line with leading white space
1
!@#$%^&*()
```

### Решение (описание макроса/клавиш)

```text
1. vim [file]
2. Esc
3. q для записи макроса
4. символ — имя регистра
5. начало строки (0) -> режим вставки (i) -> (") -> Esc -> конец строки ($) ->
   вставка в конец строки (a) -> (") -> Esc -> перейти на следующую строку (j) -> q
6. Использование макроса:

Esc — командный режим

@(имя макроса)

N@(имя макроса) — повторить макрос N раз
```

### Пояснение

- Макрос в Vim позволяет один раз записать последовательность действий и затем воспроизводить её для других строк.
- После команды `q` выбирается регистр, в который будет сохранён макрос.
- Переход к началу строки через `0` обеспечивает вставку открывающей кавычки перед первым символом строки.
- Команда `i` используется для вставки символа `"` в начало строки, а команда `a` — для добавления закрывающей кавычки в конец строки.
- Переход на следующую строку через `j` делает макрос пригодным для последовательной обработки нескольких строк.
- Вызов `@(имя макроса)` воспроизводит макрос один раз, а форма `N@(имя макроса)` позволяет сразу применить его к нескольким строкам.

### Пример запуска

```bash
cd "Paragraph 4"
vim 03_sample_strings.txt
```

## 3) Связь с оболочкой (вставка текущей даты)

### Файл

`04_insert_current_date_in_vim.txt`

### Решение (клавиши)

```text
1. Esc
2. o
3. Ctrl+R, затем =
4. strftime("%c")
5. Enter
6. Esc
```

### Пояснение

- Команда `o` создаёт новую строку ниже текущей и сразу переводит редактор в режим вставки.
- Сочетание `Ctrl+R`, затем `=` открывает ввод выражения, результат которого будет вставлен в текст.
- Выражение `strftime("%c")` формирует строку с текущими датой и временем в локальном формате системы.
- После ввода выражения нажимается `Enter`, затем `Esc` для возврата в нормальный режим.

### Пример запуска

```bash
cd "Paragraph 4"
vim 04_insert_current_date_in_vim.txt
```

## 4) Преобразование XML в Java properties

### Файлы

- `05_convert_xml_to_java_properties.txt`
- `06_sample_hdfs_properties.xml`

### Тестовые данные (XML)

```xml
<property>
 <name>dfs.namenode.name.dir</name>
 <value>/var/hdfs/name</value>
</property>

<property>
 <name>dfs.namenode.data.dir</name>
 <value>/var/hdfs/data</value>
</property>
```

### Решение (описание действий/команд)

```text
1. Открыть XML-файл в Vim.
2. Выполнить замену по всему файлу:

:%s/<property>\n\s*<name>\(.*\)<\/name>\n\s*<value>\(.*\)<\/value>\n<\/property>/\1=\2/g
```

### Пояснение

- Исходная структура XML содержит пары `<name>` и `<value>`, которые необходимо представить в формате `ключ=значение`.
- Команда подстановки `:%s/.../.../g` выполняет замену по всему файлу.
- Первая группа `\(.*\)` извлекает содержимое тега `<name>`, а вторая — содержимое тега `<value>`.
- В правой части выражения `\1=\2` эти фрагменты объединяются в формат Java properties.
- В результате запись вида `<name>dfs.namenode.name.dir</name>` и `<value>/var/hdfs/name</value>` преобразуется в строку `dfs.namenode.name.dir=/var/hdfs/name`.

### Пример запуска

```bash
cd "Paragraph 4"
vim 06_sample_hdfs_properties.xml
# В Vim выполнить: :%s/<property>\n\s*<name>\(.*\)<\/name>\n\s*<value>\(.*\)<\/value>\n<\/property>/\1=\2/g
```
