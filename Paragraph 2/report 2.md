## 1. Конвертация изображений

### Задание

Написать скрипт, который преобразует переданные в него файлы в формат PNG, используя `convert`. Для тестирования можно предварительно скачать несколько изображений из интернета.

### Решение

**Скрипт `1.sh`:**

```bash
#!/bin/bash
if [ $# -eq 0 ]; then
    echo -e "\033[31mError: No files provided.\033[0m"
    exit 1
fi

for file in "$@"; do
    if [ -f "$file" ]; then
        convert "$file" "${file%.*}-converted.png"
        echo -e "\033[32mFile $file converted to ${file%.*}-converted.png\033[0m"
    else
        echo -e "\033[31mError: File $file not found.\033[0m"
    fi
done

# wget -O image1.jpg https://example.com/image1.jpg
```

### Пояснение

- `"$@"` обозначает полный список аргументов, переданных скрипту, при этом каждый аргумент обрабатывается как отдельное значение.
- Проверка `if [ $# -eq 0 ]` позволяет определить ситуацию, когда пользователь не указал ни одного файла для обработки.
- Условие `-f "$file"` проверяет существование обычного файла с указанным именем.
- Команда `convert` выполняет преобразование исходного изображения в формат PNG.
- Конструкция `"${file%.*}"` удаляет расширение исходного файла и позволяет сформировать новое имя результата.

### Пример запуска

```bash
chmod +x 1.sh
./1.sh image1.jpg image2.jpg
```

## 2. Мониторинг

### Задание

Скрипт должен вывести показания всех температурных датчиков, поддерживаемых ядром Linux.

Необходимо найти файлы в `/sys`, имена которых начинаются на `temp` и заканчиваются на `input`, а затем вывести их содержимое в удобном формате.

### Решение

```bash
#!/bin/bash
find /sys -type f -name "temp*input" 2>/dev/null | while read sensor_file; do
    echo "Sensor: $sensor_file"
    awk '{printf("%.1f°C\n",$1/1000)}' "$sensor_file"
    echo ""
done
```

### Пояснение

- `find /sys -type f -name "temp*input"` выполняет поиск файлов температурных датчиков в виртуальной файловой системе `sysfs`.
- Перенаправление `2>/dev/null` исключает из вывода сообщения об ошибках доступа.
- Конструкция `| while read sensor_file; do ... done` организует последовательную обработку найденных путей.
- `awk '{printf("%.1f°C\n",$1/1000)}'` преобразует значение температуры из тысячных долей градуса в стандартную запись с одной цифрой после запятой.
- Вывод имени файла датчика перед значением температуры делает результат более наглядным и позволяет соотнести показание с источником данных.

### Пример запуска

## 3. Параллельная обработка

### Задание

Переписать скрипт конвертации таким образом, чтобы несколько изображений обрабатывались параллельно.

Следует использовать фоновый запуск через `&` и команду `wait`.

### Решение

```bash
#!/bin/bash
if [ $# -eq 0 ]; then
    echo -e "\033[31mError: No files provided.\033[0m"
    exit 1
fi

# Массив для хранения ID процессов
pids=()
files=()

for file in "$@"; do
    if [ -f "$file" ]; then
        if [[ "$file" =~ \.(jpg|jpeg)$ ]]; then
            convert "$file" "${file%.*}-converted.png" &
            pids+=($!)
            files+=("$file")
        else
            echo -e "\033[31mError: $file is not a valid image file.\033[0m"
        fi
    else
        echo -e "\033[31mError: File $file not found.\033[0m"
    fi
done

for i in "${!pids[@]}"; do
    wait ${pids[$i]}
    if [ $? -eq 0 ]; then
        echo -e "\033[32mConversion completed for file: ${files[$i]}.\033[0m"
    else
        echo -e "\033[31mError occurred during conversion for file: ${files[$i]}.\033[0m"
    fi
done
```

### Пояснение

- Символ `&` переводит выполнение команды `convert` в фоновый режим, благодаря чему цикл может немедленно перейти к следующему файлу.
- Переменная `$!` содержит идентификатор последнего запущенного фонового процесса.
- Массив `pids` используется для хранения PID запущенных процессов, а массив `files` — для связи каждого процесса с соответствующим исходным файлом.
- Команда `wait` приостанавливает выполнение скрипта до завершения конкретного фонового процесса.
- После `wait` анализируется код возврата `"$?"`, что позволяет определить, завершилась ли конвертация успешно.

### Пример запуска

```bash
cd "Paragraph 2"
./3.sh image1.jpeg image2.jpg
```

## 4. Конвейер

### Задание

Скрипт должен:

1. найти JPG-файлы;
2. преобразовать их в PNG;
3. организовать работу так, чтобы поиск и конвертация были объединены в конвейер.

Также требуется ответить, будет ли такой вариант быстрее параллельного при обработке большого количества файлов.

### Решение

**Скрипт:**

```bash
#!/bin/bash
if [ $# -eq 0 ]; then
    echo -e "\033[31mError: No directory provided.\033[0m"
    exit 1
fi

if [ ! -d "$1" ]; then
    echo -e "\033[31mError: $1 is not a valid directory.\033[0m"
    exit 1
fi

find "$1" -type f -iname "*.jpg" | while read file; do
    convert "$file" "${file%.*}.png"
    echo -e "\033[32mConverted $file to ${file%.*}-converted.png\033[0m"
done
```

### Пояснение

- Проверка `if [ $# -eq 0 ]` гарантирует, что пользователь передал путь к каталогу.
- Условие `[ ! -d "$1" ]` позволяет убедиться, что указанный аргумент действительно является директорией.
- Команда `find "$1" -type f -iname "*.jpg"` ищет все JPG-файлы в указанном каталоге и его вложенных директориях.
- Конвейер `find ... | while read file; do ... done` передаёт найденные имена файлов непосредственно в цикл обработки.
- Команда `convert` создаёт PNG-файл на основе каждого найденного JPG-изображения.
- В данном скрипте имя выходного файла в команде `convert` и имя, выводимое в сообщении `echo`, различаются; это связано с особенностью исходного кода и не изменяется.

### Ответ на вопрос о производительности

Для обработки большого числа файлов конвейерный вариант, как правило, не превосходит параллельный по скорости, поскольку в нём файлы преобразуются последовательно. Параллельная схема эффективнее в ситуациях, когда система может одновременно выполнять несколько операций `convert`. Вместе с тем конвейерный подход удобен своей простотой, наглядностью и тем, что обработка начинается сразу по мере нахождения файлов без предварительного накопления полного списка.

### Пример запуска

```bash
cd "Paragraph 2"
./4.sh .
```
