## Ход работы

## 1) Стандартные права доступа для файлов и директорий

### Файл

`01_set_standard_permissions.sh`

### Решение

```bash
#!/bin/bash

find . -type f -exec chmod 644 {} \;  # для файлов
find . -type d -exec chmod 755 {} \;  # для директорий
```

### Пояснение

- `find . -type f -exec chmod 644 {} \;` последовательно находит обычные файлы в текущем каталоге и назначает им права `644`, то есть `rw-r--r--`.
- Такой режим оставляет владельцу право чтения и записи, а группе и остальным пользователям — только чтение.
- `find . -type d -exec chmod 755 {} \;` применяется к директориям и устанавливает права `rwxr-xr-x`.
- Для каталогов наличие бита выполнения имеет практическое значение: он позволяет переходить в директорию и обращаться к объектам внутри неё.

### Пример запуска

```bash
cd "Paragraph 5"
./01_set_standard_permissions.sh
```

## 2) Символьные ссылки

### Файл

`02_create_symbolic_link.sh`

### Решение

```bash
#!/bin/bash

mkdir -p dir
ln -sfn dir my-dir

# -s — создать символьную ссылку
# -f — заменить существующий объект с тем же именем
# -n — не разыменовывать ссылку на директорию при замене
```

### Пояснение

- `mkdir -p dir` создаёт каталог `dir`; ключ `-p` позволяет выполнять команду повторно без ошибки, если каталог уже существует.
- `ln -sfn dir my-dir` создаёт символьную ссылку `my-dir`, указывающую на `dir`.
- Ключ `-s` задаёт создание именно символьной, а не жёсткой ссылки.
- Ключ `-f` позволяет обновить ссылку, если объект с таким именем уже существует.
- Ключ `-n` важен при работе со ссылкой на каталог, так как предотвращает разыменование существующей ссылки во время перезаписи.

### Пример запуска

```bash
cd "Paragraph 5"
./02_create_symbolic_link.sh
```

## 3) Точки монтирования

### Файл

`03_show_mount_points.sh`

### Решение

```bash
#!/bin/bash

column -t /proc/mounts
```

### Пояснение

- Файл `/proc/mounts` содержит сведения о смонтированных файловых системах в текущей системе.
- Команда `column -t /proc/mounts` форматирует содержимое этого файла по столбцам без лишнего конвейера.
- В таком виде проще различать устройство, точку монтирования, тип файловой системы и параметры подключения.

### Пример запуска

```bash
cd "Paragraph 5"
./03_show_mount_points.sh
```

## 4) Именованные каналы (FIFO)

### Файлы

- `04_read_messages_from_fifo.sh`
- `05_write_messages_to_fifo.sh`

### Решение

**Скрипт чтения:**

```bash
#!/bin/bash

PIPE_NAME="message_pipe"

if [ ! -p "$PIPE_NAME" ]; then
    echo "$PIPE_NAME does not exist"
    exit 1
fi

while true; do
    if read -r message <"$PIPE_NAME"; then
        echo "$(date) - $message"
    fi
done
```

**Скрипт записи:**

```bash
#!/bin/bash

PIPE_NAME="message_pipe"

echo "Hello, world!" >"$PIPE_NAME"
sleep 1
echo "This is a test message." >"$PIPE_NAME"
sleep 1
echo "Another message!" >"$PIPE_NAME"
```

### Пояснение

- Перед началом работы необходимо создать именованный канал командой `mkfifo message_pipe`.
- Переменная `PIPE_NAME` делает имя канала единообразным в обоих скриптах.
- Проверка `[ ! -p "$PIPE_NAME" ]` убеждается, что объект существует и действительно является FIFO.
- Цикл `while true` поддерживает постоянное ожидание новых сообщений в канале.
- Команда `read -r message <"$PIPE_NAME"` считывает очередную строку, поступившую в канал, и сохраняет её в переменную `message`.
- Конструкция `echo "$(date) - $message"` выводит принятое сообщение вместе с текущими датой и временем.
- Во втором скрипте сообщения передаются в канал поочерёдно, а `sleep 1` создаёт между ними небольшую паузу.

### Пример запуска

1. В первом терминале создать канал и запустить читатель:

```bash
mkfifo message_pipe
./04_read_messages_from_fifo.sh
```

2. Во втором терминале запустить отправку сообщений:

```bash
./05_write_messages_to_fifo.sh
```

3. В первом терминале появятся строки вида:

- `<текущая дата> - Hello, world!`
- `<текущая дата> - This is a test message.`
- `<текущая дата> - Another message!`
